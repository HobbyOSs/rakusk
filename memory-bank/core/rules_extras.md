# Rakusk Development Rules Extras

## テストと品質管理

### 回帰テストの義務付け
- コードの改修（機能追加、リファクタリング、バグ修正など）を行った後は、必ず既存のテストスイートを実行して回帰テストを行わなければなりません。
- テストが失敗した状態でのコミットや、タスクの完了報告は禁止されています。

### テスト実行方法
- プロジェクトルートで `make test` コマンドを実行してください。
- 内部的には `raku -Ilib t/assembler.t` が実行され、TAP 形式で結果が出力されます。

## コーディング規約

- Raku の慣習に従った命名規則を使用してください。
- 各パス（Pass1, Pass2）の独立性を保ち、共通の AST を介してデータをやり取りするようにしてください。
