## 最近の修正（技術的詳細）
- **命令選択精度の向上**: `Actions.rakumod` において `extension` (group opcode) の有無を厳密にチェックするようにし、`reg-imm` 形式への誤マッチングを防止。
- **エンコードの正規化**: nask/Intel 互換性を高めるため、プレフィックス順序（`0x67` -> `0x66`）と `IMUL` (2-operand) の ModR/M 生成ロジックを修正。
- **負数処理の安定化**: `pack-le` で明示的なビットマスクを適用し、負のディスプレースメントのエンコード不備を解消。
