# Active Context

## 最近の変更点
- **Pass 1 / Pass 2 のリファクタリング**:
    - コードの肥大化を防ぐため、`Rakusk::Pass1` と `Rakusk::Pass2` を AST の構造（Instruction, Pseudo, Statement）に合わせてサブモジュール化。
    - 各サブモジュールを `role` として定義し、`Core` クラスで合成する構成を採用。
    - 外部インターフェースを維持したまま、メンテナンス性を向上。
- `gosk` の `day02.t` (Hello OS 3 ブートセクタ) の完全な移植とパス。
- **16ビットリアルモード対応の強化**:
    - 16ビットアドレッシングモード (`[SI]`, `[BX+DI]` 等) の ModR/M エンコードを `Pass2` に実装。
    - セグメントレジスタ (`MOV DS, AX` 等) のエンコードを実装。
- **特化型命令 (short-imm) の導入**:
    - `CMP AL, imm8` 等の ModR/M を使わない最短形式をサポート。

## 現在の焦点
- ブートローダー作成に必要な基本機能の網羅。
- FAR ジャンプ（セグメント付きアドレス）のサポートに向けた AST の拡張。

## 次のステップ
- **FAR ジャンプの AST 対応**: `SegmentedAddress` ノードの定義とパース、パス1でのサイズ計算。
- **SIB バイトのサポート**: 32ビットアドレッシングモードへの拡張。
- **512バイトブートセクタの出力支援**: パディングやシグネチャの自動付与。
