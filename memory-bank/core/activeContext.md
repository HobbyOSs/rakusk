## 得られた知識
- **BDO（マルチパス最適化）における「楽観的初期化」の罠**: 
    - 全ての JMP を 2バイトと仮定すると、短縮不可能な間接ジャンプ（最低4バイト以上）が破壊的な影響を受ける。初期化対象は、オペランドが `Immediate`（ラベル等）である相対ジャンプに限定する必要がある。
- **COFF セクションのアドレス計算**: 
    - COFF形式では各セクションがアドレス 0 から始まるため、アセンブラ内部の PC もセクションごとにリセットしなければ、アライメント計算がズレる。
- **Pass 1 と Pass 2 の不一致によるシンボルズレ**: 
    - 命令サイズが Pass 1 (サイズ見積もり) と Pass 2 (エンコード) で異なると、バイナリデータ自体が一致していてもシンボルテーブル内のアドレスがずれる原因となる。
