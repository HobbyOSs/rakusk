# Active Context

## 最近の変更点
- **Pass 1 / Pass 2 の大規模リファクタリング**:
    - `Pass2::Instruction` に共通のエンコードロジック（prefix判定、opcode生成、ModR/M・SIB・Disp生成、即値生成）を抽出。
    - `Pass1::Instruction` が `Pass2::Instruction` を取り込み、実際のエンコード結果からサイズを算出する構成に変更。これによりロジックの完全な同期を実現。
    - `Pass1` のコード量を約80%削減し、メンテナンス性を大幅に向上。
- **Grammar/Actions の整理**:
    - `try-special-variant` メソッドを導入し、最適化バリアント（imm8算術演算等）の選択ロジックを分離・整理。
- `gosk` の `day02.t` (Hello OS 3 ブートセクタ) の完全な移植とパス。
- **16ビットリアルモード対応の強化**:
    - 16ビットアドレッシングモード (`[SI]`, `[BX+DI]` 等) の ModR/M エンコードを `Pass2` に実装。
    - セグメントレジスタ (`MOV DS, AX` 等) のエンコードを実装。
- **特化型命令 (short-imm) の導入**:
    - `CMP AL, imm8` 等の ModR/M を使わない最短形式をサポート。

## 現在の焦点
- **day04 以降の移植**:
    - C言語との連携や、より高度なOS機能の実装に向けたアセンブラの拡張。
    - ELF 出力やオブジェクトファイルのサポート（必要に応じて）。

## 最近の変更点
- **day03 (harib00a〜harib00i) の完全な移植とパス**:
    - 16ビットリアルモードから32ビットプロテクトモードへの移行に必要な命令セットを網羅。
    - **32ビットアドレッシングのサポート**: 16ビットモード下での `[ESI]` 形式などのエンコード、SIBバイト、Address Size Prefix (67h) の実装。
    - **エンコード最適化**: `OR EAX, 1` 等の命令で `reg-imm8` (83h) 形式を選択するサイズベースのバリアント優先順位付けの実装。
    - **コントロールレジスタ対応**: `MOV EAX, CR0` 等のエンコードを `reg-cr` 形式として正しく処理。
    - **不足命令の追加**: `LGDT`, `IN`, `OUT`, `AND`, `OR`, `SUB`, `IMUL`, `JNC`, `JAE` 等。
    - `ALIGNB` 疑似命令の Pass 2 でのバイナリ出力実装。

## 次のステップ
- **512バイトブートセクタの出力支援**: パディングやシグネチャの自動付与。
- **エラーメッセージの改善**: 無効なレジスタの組み合わせ等に対するエラー報告の強化。
